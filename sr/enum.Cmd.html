Stats

Aggregate per-process statistics across all SRUM tables.

Builds a merged timeline and summarises each app’s CPU cycles, bytes, active intervals, and heuristic flags. Output sorted by flag_count desc, then total_background_cycles desc. Best-effort: always exits 0.

Fields

§path: PathBuf

Path to SRUDB.dat (or a forensic copy of it).

§resolve: bool

Resolve app_id to names from SruDbIdMapTable.

§format: OutputFormat

Output format (json, csv, or ndjson).

§

Sessions

Derive user keyboard sessions from the SRUM timeline.

A session is a contiguous run of timestamps where user_present: true. A gap > 2 hours between timestamps starts a new session. Best-effort: always exits 0.

Fields

§path: PathBuf

Path to SRUDB.dat (or a forensic copy of it).

§format: OutputFormat

Output format (json, csv, or ndjson).

§

Gaps

Detect temporal gaps in SRUM records — identifies system-off periods and potential targeted record deletion.

Analyses timestamps from the merged timeline to detect two kinds of suspicious gaps:

  • system_off: ALL tables have a gap at the same time window.
  • selective_gap: Only ONE specific table has a gap while others have records.

Best-effort: always exits 0 even for nonexistent files.

Fields

§path: PathBuf

Path to SRUDB.dat (or a forensic copy of it).

§threshold_hours: u64

Minimum gap size in hours to report (default: 2).

§format: OutputFormat

Output format (json, csv, or ndjson).

§

Hunt

Hunt for specific forensic patterns across all SRUM tables.

Filters the merged timeline to records matching a named heuristic signature. Best-effort: always exits 0 even for missing files.

Fields

§signature: HuntSignature

Named forensic pattern to hunt for.

§path: PathBuf

Path to SRUDB.dat.

§resolve: bool
§format: OutputFormat
§

Compare

Compare two SRUDB.dat files and surface what changed between them.

Detects new processes, departed processes, and processes whose behaviour changed (new heuristic flags, significant byte-count deltas).

Fields

§baseline: PathBuf

Baseline SRUDB.dat (before the incident).

§suspect: PathBuf

Suspect SRUDB.dat (after the incident).

§resolve: bool

Resolve app_id/user_id to names for process matching.

§format: OutputFormat
§

Timeline

Merge all SRUM tables into a single chronological timeline.

Reads network, apps, connectivity, energy, notifications, and focus records, injects a source_table field on each entry, and sorts by timestamp. Apps records are automatically flagged with heuristic signals. Tables that are absent or unreadable are silently skipped.

Fields

§path: PathBuf

Path to SRUDB.dat (or a forensic copy of it).

§resolve: bool

Resolve app_id and user_id to names from SruDbIdMapTable.

Adds app_name and user_name fields to all records that carry those integer IDs.

§format: OutputFormat

Output format (json or csv).

§

Metadata

Extract metadata from a SRUDB.dat file: SHA-256 hash, table enumeration, record counts, temporal span, and Windows version hint.

Fields

§path: PathBuf
§format: OutputFormat
