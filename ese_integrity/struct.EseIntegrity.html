pub struct EseIntegrity<'a> {
    pub(crate) data: &'a [u8],
}
Expand description

Read-only forensic analyser for a raw ESE database byte buffer.

Operates directly on raw bytes so it can detect anomalies that would prevent normal parsing (bad checksums, missing pages, truncation). analyse short-circuits after any Critical finding: a truncated file makes page-level checks meaningless.

pub fn analyse(&self) -> Vec<EseStructuralAnomaly>

Run all checks and return every detected anomaly.

Returns an empty Vec for a structurally sound database. Short-circuits after any Critical finding (e.g. TruncatedDatabase).

pub fn check_layout(&self) -> Vec<EseStructuralAnomaly>

Check for database layout problems (truncation).

Returns TruncatedDatabase if the buffer is smaller than ESE_MIN_BYTES or shorter than the declared page count implies.

Source

pub fn check_pages(&self) -> Vec<EseStructuralAnomaly>

Check page-level integrity (XOR checksums).

For each data page, verifies the stored XOR checksum at offset 0 against the computed value. Pages where the stored checksum is 0 are treated as “unchecked” and skipped (common for empty/synthetic pages). Handles both legacy XOR and Vista+ ECC formats via ese_core::verify_page_checksum().
